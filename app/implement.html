Selecting a Runtime Container

camunda BPM is a flexible framework for workflow and process automation. It's core is a native BPMN 2.0 process engine that runs inside the Java Virtual Machine. Camunda gives you the coice of your favorite runtime, be it a standalone Java Virtual Machine, a Servlet Container, a Java EE Application Server or even OSGi.

Use unique features and innovations delivered by state of the art containers like JBoss AS 7 and combine them with our rock-solid BPM process engine.

(Click on the icons above to discover the runtimes)

camunda BPM in a standalone JVM

The JVM is an aswesome platform. Integrating camunda BPM with nothing but the bare metal gives you maximal flexibility when it comes to your integration requirements. Yet, it already gives you advanced features like integration with the JMX management container, asynchronous processing and much more. And it's great for unit testing!

Setting up a Process Application Project

A process application is a Java application that interacts with BPMN 2.0 processes. This means that part of the behavior and the functionality of the process application is described by one or multiple BPMN 2.0 process models. On this page you can discover the necessary steps that allow you to turn a Java Application into a Process Application.
